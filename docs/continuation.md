## What's next

Three of the four surfaces are **landed** тАФ the Hono API trunk (`packages/api-server`), the CLI (`apps/cli`, compiled `ez` binary), and MCP (`/mcp` on the trunk). The Web UI is the one unbuilt surface; its **architecture is now decided** (ADR 0027тАУ0033, Accepted 2026-05-30) and implementation is the immediate focus.

**Web UI surface-architecture тАФ DECIDED (ADR 0027тАУ0033, 2026-05-30).** Redone from scratch via a 36-agent exhaustive review + 5 red-teamers + a cross-model Codex ADR pass; superseded ADR 0005 (Next.js) and ADR 0004 (BlockNote-as-final). The shape:

- **Topology (0027):** the Hono trunk is the top-level server (`@hono/node-server` **v1** тАФ `getRequestListener` + raw `ws`; deliverable #2 retired the earlier "v2 `serve()` prerequisite" framing) тАФ Vite + React SPA as static assets, JSON-RPC + `/auth` + `/mcp`, event-rendered static published HTML, embedded Hocuspocus on one port. No Next.js (`output:export` тКе `standalone`; Next can't host the collab WS).
- **Routing (0028):** TanStack Router; one typed-client seam (`packages/api-client`); raw `hc<AppType>` forbidden outside it.
- **API shape (0029, substrate reversed 2026-05-30; MIGRATED 2026-05-30):** **code-first** per-route factories тАФ `hono-openapi` `describeRoute` + `validator` on `hono/factory` handlers, composed via `.route()` chaining (the `openapiRoutes([...] as const)` tuple is **retired**; `tsc` spikes showed `.route()` preserves the `hc` merge). Capability schemas reused directly at `validator` from `@editorzero/schemas` (SSOT, ADR 0034 тАФ no restated route schema); typed error arm = explicit per-route `c.json` returns. **Landed** (see Recent history 2026-05-30): all routes + 5 domain index sub-apps + the `app.ts` trunk are code-first; `@hono/zod-openapi` dropped. Registry-driven *generation* of the route files is still future (the files are hand-maintained today, parity-guarded by the CLIтЖФserver test).
- **Auth (0030):** Better Auth in-trunk, same-origin тЖТ `SameSite=Lax`, zero framework adapter.
- **Editor (0031):** bootstrap on BlockNote, then eject to Tiptap v3 + owned thin block layer (clean-start), fused with track-changes.
- **Version history + track-changes (0032):** build ourselves on Yjs snapshots + `prosemirror-changeset` (restore appends a corrective update; anchors are Yjs `RelativePosition`; tracked *agent* edits are the differentiator).
- **Testing + RPC (0033):** typed error envelope at the route boundary, real-browser editor tests, `ui` parity cell fails the build.
