# ADR 0010 — SSO: Better Auth + @better-auth/sso

**Status:** Accepted (post-red-team)
**Date:** 2026-04-17
**Deciders:** @numman

## Context
Multi-tenant SSO: one deployment, many workspaces, each with its own OIDC and/or SAML 2.0 IdP. Library integration (no bolt-on IdP sidecar). Agent tokens and human PATs go through the same auth spine and feed into the `Principal` model (ADR 0016).

## Options considered
- **Better Auth + `@better-auth/sso`** (TS) — OIDC via `jose` + SAML via `samlify`; multi-tenant `providerId` with per-workspace config; library-level mount into Hono routes. 1.5 release closed CVE-2025-47949 via `samlify ≥ 2.12` and added InResponseTo replay protection + algorithm allowlist + clock-skew config. Active daily commits. MIT.
- **Auth.js v5** — no native SAML; wraps BoxyHQ/SSOReady (bolt-on pattern we're avoiding).
- **Lucia** — deprecated March 2025.
- **openid-client + passport-saml** — viable but multi-tenant registry DIY; passport-saml CVE history noisy.

## Decision
**Better Auth + `@better-auth/sso`.** Pin `samlify >= 2.12`. Per-workspace `providerId` with either `oidcConfig` or `samlConfig`.

### Tokens and principals
- **Human session tokens:** issued by Better Auth after SSO login; short-lived; attached to HTTP cookies.
- **PATs (human):** long-lived, revocable, scoped to the human principal; issued via `/settings/tokens` UI.
- **Agent tokens (ADR 0016):** long-lived, revocable, workspace-scoped; minted through Better Auth's token system with a distinct `principal.kind = 'agent'` tag so audit, rate limits, and permissions route through the agent-aware code paths.
- All three token shapes flow through a single validation middleware; the resulting `Principal` (ADR 0016) is what every capability (ADR 0015) sees.

### MCP audience coupling
MCP (ADR 0009) is an OAuth 2.1 resource-server. For custom-domain tenants, Better Auth is configured with per-tenant canonical URLs so the token it mints carries the correct `aud`. Changing a tenant's canonical URL triggers token re-issuance.

## Consequences
- OIDC + SAML first-class, multi-tenant, from day one.
- Single auth spine for humans and agents; uniform audit (ADR 0019) and rotation paths.
- Track Better Auth CVE / release cadence; pin versions.
- Better Auth is MIT; AGPL-compatible in the consuming direction.

## Revisit triggers
- Better Auth maintainership stalls (release cadence below one per quarter) or a sustained CVE with slow upstream response.
- A critical enterprise IdP quirk cannot be handled through Better Auth's abstraction and requires lower-level ACS/metadata wiring.
- We become an OAuth 2.1 Authorization Server for external clients (beyond the MCP resource-server flow) — add `fosite`-equivalent behavior alongside Better Auth.
